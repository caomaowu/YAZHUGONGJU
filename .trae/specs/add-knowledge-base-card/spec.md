# 知识库卡片（资料库）Spec

## Why
当前工具缺少统一的资料存放与远程查看入口，书籍/文档/图片等资料分散，检索与阅读体验不一致。新增“知识库”卡片用于集中管理与预览资料，提升日常查阅效率与界面体验。

## What Changes
- 新增左侧导航/工作台入口卡片：知识库（Knowledge Base）
- 新增知识库页面：文件列表（卡片/画廊）+ 搜索筛选 + 上传入口 + 预览阅读
- 后端新增知识库文件存储与元数据持久化：文件落盘 + library.json 记录元数据
- 后端新增知识库 API：上传/列表/删除/下载/预览（流式或直连）
- UI 统一为 Lavender 风格的“玻璃拟态 + 柔和渐变 + 动效”卡片式体验

## Impact
- Affected specs: 资料存储、远程查看、权限控制、预览阅读体验
- Affected code:
  - 前端：路由/侧边栏入口、知识库页面与组件（预览弹窗/抽屉、上传面板、筛选器）
  - 后端：Express 新增知识库路由、文件上传与静态/流式读取、元数据读写

## ADDED Requirements
### Requirement: 知识库入口
系统 SHALL 在左侧导航（或工作台）提供“知识库”入口，用户点击后进入知识库页面。

#### Scenario: 进入知识库页面
- **WHEN** 用户点击“知识库”
- **THEN** 系统展示知识库页面，并显示资料列表与搜索区域

### Requirement: 资料上传与元数据
系统 SHALL 允许有权限的用户上传资料文件，并为每个文件维护元数据（至少包含：id、原始文件名、存储文件名/路径、类型、大小、上传时间、上传者、可选标签/分类、可选描述）。

#### Scenario: 成功上传
- **WHEN** 用户选择文件并提交上传
- **THEN** 系统保存文件到服务器存储目录，并在元数据存储中新增记录
- **AND** 前端列表立即可见该资料

### Requirement: 资料浏览与检索
系统 SHALL 提供资料列表视图与搜索能力，用户可按关键字检索文件名与描述，并可按类型/分类/标签进行筛选。

#### Scenario: 搜索命中
- **WHEN** 用户输入关键字并触发搜索
- **THEN** 列表仅显示匹配资料，支持清空恢复

### Requirement: 在线预览与下载
系统 SHALL 支持对常见资料类型进行在线预览，并提供下载原文件能力。

#### Supported Preview（MVP）
- PDF：在线预览
- 图片（png/jpg/jpeg/webp/gif）：在线预览（画廊/缩放）
- Markdown/TXT：在线渲染/阅读

#### Scenario: 预览资料
- **WHEN** 用户点击某条资料的“预览”
- **THEN** 系统在预览弹窗/抽屉中展示内容
- **AND** 提供关闭、下载等操作

#### Scenario: 下载资料
- **WHEN** 用户点击“下载”
- **THEN** 浏览器下载原始文件名的文件

### Requirement: 远程查看
系统 SHALL 允许在同一网络环境下，通过访问部署该系统的 Web 地址实现远程查看知识库资料（在鉴权通过后可预览/下载）。

#### Scenario: 远程访问
- **WHEN** 用户在另一台设备上访问系统地址并登录
- **THEN** 可进入知识库并预览/下载资料

### Requirement: 权限与安全（MVP）
系统 SHALL 复用现有登录鉴权机制保护知识库 API；未登录用户不可访问资料内容。

#### Default Policy（可在实现阶段按现有权限体系落地）
- 查看/下载：登录用户可用
- 上传/删除：管理员或具备管理权限的角色可用

无。
