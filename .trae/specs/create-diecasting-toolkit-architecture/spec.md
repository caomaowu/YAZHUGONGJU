# 压铸工艺小工具集（Web 工具集）技术方案 Spec

## Why

压铸工艺中常用的 PQ² 图、参数联动计算、模板管理等能力分散在不同工具中，缺少统一、跨平台且可扩展的工作台。本方案用 Web 技术快速交付一套可持续扩展的工具集，并为后续桌面打包预留路径。

## What Changes

- 建立“工具集工作台”基础架构：工具注册中心、路由与布局、共享状态与数据互通机制
- 建立统一 UI 视觉语言：配色、渐变、高亮、圆角卡片、柔和阴影、三栏布局规范
- 实现 PQ² 图工具：ECharts 可视化、参数输入与联动计算、导出图片/数据
- 实现模板管理：本地文件读写（File System Access API）、最近使用、工作区模式（选择目录作为工作区）
- 提供扩展接口：新增工具只需注册即可接入导航与共享状态
- **BREAKING**：无（新项目初始化阶段）

## Impact

- Affected specs: 工具注册与导航、共享状态、PQ² 可视化、模板文件系统、本地持久化
- Affected specs: UI 视觉语言与布局规范、主题与样式 Token
- Affected code: 前端应用（React + TypeScript + Vite + Ant Design + ECharts），未来可选桌面壳（Tauri）集成层

## ADDED Requirements

### Requirement: UI 视觉与布局风格（Lavender Pastel）

系统 SHALL 提供明亮、柔和、轻量、精致、亲和且带轻卡通感的专业 UI，并通过统一主题与组件规范保持一致性。

#### Scenario: 统一配色与对比控制

- **WHEN** 用户浏览任意工具页面
- **THEN** 页面遵循温和低对比配色：主色为柔紫/淡粉紫/浅薰衣草紫（示例：#8C7CF0、#C6B9FF），辅色使用柔绿/柔黄/淡橙作强调色，背景为大面积近白浅灰白

#### Scenario: 模块高亮与“云朵感”质感

- **WHEN** 页面展示模块或信息分组
- **THEN** 模块以紫→粉的轻渐变色块或柔和容器作为背景，并使用柔和浅阴影营造悬浮感，无高对比硬阴影与硬边框

#### Scenario: 三栏布局与柔软卡片信息分组

- **WHEN** 用户进入工作台主界面
- **THEN** 典型布局为三栏：左侧图标化导航、中间主要内容（卡片 + 信息区）、右侧附加板块；整体采用超大留白、圆角卡片、清晰信息分组（大标题 + 小副标题 + 极简 icon + 轻量进度条等）

### Requirement: 工具集工作台

系统 SHALL 提供一个工具集工作台，支持多个工具在同一应用内集成，并通过统一导航访问。

#### Scenario: 访问与切换工具

- **WHEN** 用户打开应用并从导航选择某个工具
- **THEN** 系统展示该工具页面，并保留工具间共享状态（按设计的作用域）

### Requirement: 工具注册中心

系统 SHALL 提供工具注册中心，用于声明工具的元信息、路由入口、权限/特性开关（预留）、以及与共享状态的连接方式。

#### Scenario: 新增工具接入

- **WHEN** 开发者新增一个工具并完成注册
- **THEN** 工具自动出现在导航中，并可获得共享状态与公共能力（主题、导出、错误提示等）

### Requirement: 共享状态与数据互通

系统 SHALL 提供共享状态层，允许工具之间安全地共享必要数据（例如材料参数、机台信息、默认单位等），并支持版本化/命名空间以避免冲突。

#### Scenario: 工具间复用参数

- **WHEN** 用户在工具 A 中保存一组全局参数
- **THEN** 工具 B 可读取同名命名空间下的参数并用于默认值或计算

### Requirement: PQ² 图可视化

系统 SHALL 提供 PQ² 图工具，支持输入关键工艺参数并绘制 PQ² 相关曲线/区域，支持交互（缩放、悬浮提示、图例切换）。

#### Scenario: 绘图成功

- **WHEN** 用户输入/选择完整的 PQ² 所需参数并点击“绘制”
- **THEN** 系统生成 PQ² 图并展示关键点/线，且参数变化能触发重算与更新

### Requirement: 参数联动计算

系统 SHALL 在 PQ² 图工具内提供参数联动计算能力，支持输入项校验、单位显示（预留）、以及可追溯的计算结果展示。

#### Scenario: 输入校验与结果更新

- **WHEN** 用户修改任意输入参数
- **THEN** 系统即时校验并在有效时更新相关计算结果与图表

### Requirement: 模板管理（本地文件）

系统 SHALL 支持模板文件的本地读写，允许用户选择模板文件或工作区目录，并记住最近使用记录。

#### Scenario: 选择模板文件并加载

- **WHEN** 用户选择本地模板文件
- **THEN** 系统读取并加载模板内容到 UI，且在下次启动时可从最近使用中快速打开（在浏览器权限允许范围内）

#### Scenario: 工作区模式

- **WHEN** 用户选择一个目录作为工作区
- **THEN** 系统在该目录下读取/保存模板，并提供浏览、创建、重命名、删除（受 API 能力限制）等操作入口

### Requirement: 导出与分享（基础）

系统 SHALL 支持将 PQ² 图导出为图片（PNG/SVG，按可用能力）以及导出当前参数为 JSON。

#### Scenario: 导出图片

- **WHEN** 用户点击导出按钮
- **THEN** 系统生成并下载/保存导出文件，文件名包含工具名与时间戳
