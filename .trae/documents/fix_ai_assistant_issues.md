# AI 助手问题修复计划

本计划旨在解决 `问题.md` 中列出的四个核心问题，包括输入框定位、内容抖动、滚动策略和文本渲染异常。

## 1. 问题分析

1.  **输入框定位问题**：生成内容时输入框被顶出可视区域。
    *   **原因**：`.aiShell` 或 `.aiMain` 容器的高度约束不足，导致内容撑开整个容器而非在 `.aiMessages` 内部滚动。
    *   **对策**：加强 CSS 布局约束，确保 flex 容器正确处理溢出。

2.  **内容区域抖动**：生成过程中内容上下跳动。
    *   **原因**：`AiMarkdown` 组件中 `ReactMarkdown` 的 `components` 属性在每次渲染时重新定义，导致 DOM 树完全重建（Reflow）。
    *   **对策**：使用 `useMemo` 缓存 `components` 配置，或将其移至组件外部。

3.  **滚动策略粗暴**：强制滚动到底部，打断用户阅读。
    *   **原因**：流式更新时无条件调用滚动逻辑，且缺乏对用户当前滚动位置的判断。
    *   **对策**：引入"智能粘底"逻辑，仅在用户当前处于底部附近时才自动滚动。

4.  **文本渲染异常**：字符重复、乱码。
    *   **原因**：SSE 流解析逻辑使用 `split('\n\n')` 可能在某些边界条件下处理不当，或状态更新逻辑有误。
    *   **对策**：重构 SSE 解析器，采用更稳健的逐行解析状态机模式。

## 2. 修改步骤

### 步骤 1: 修复 UI 布局 (输入框定位)
*   **文件**: `src/pages/AIAssistantPage.css`
*   **修改**:
    *   确保 `.aiShell` 和 `.aiMain` 具有明确的 `height: 100%` 和 `overflow: hidden`。
    *   确保 `.aiMessages` 在 flex 布局中能正确滚动。

### 步骤 2: 优化渲染性能 (内容抖动)
*   **文件**: `src/components/ai/AiMarkdown.tsx`
*   **修改**:
    *   将 `ReactMarkdown` 的 `components` 属性定义提取到组件外或使用 `useMemo` 包裹。
    *   确保代码块渲染组件稳定。

### 步骤 3: 优化滚动体验
*   **文件**: `src/pages/AIAssistantPage.tsx`
*   **修改**:
    *   改进 `isAtBottom` 的判定逻辑。
    *   在 `applyDelta` 中，仅当 `isAtBottom` 为 true 时调用滚动。
    *   发送新消息时强制滚动。

### 步骤 4: 重构流式解析 (文本异常)
*   **文件**: `src/pages/AIAssistantPage.tsx`
*   **修改**:
    *   重写 `streamChat` 中的 `while` 循环解析逻辑。
    *   实现一个基于行的解析器，正确处理 `data:` 前缀和多行数据拼接。
    *   确保 `TextDecoder` 正确处理 UTF-8 边界。

## 3. 验证计划
*   **布局验证**: 检查长对话生成时，输入框是否固定在底部。
*   **抖动验证**: 观察生成过程中文本是否平滑追加，无闪烁。
*   **滚动验证**: 在生成过程中向上滚动查看历史消息，确认不被强制拉回底部；手动滚到底部后，确认能恢复自动跟随。
*   **内容验证**: 确认中文输出无乱码、无重复段落。
