# 压铸工具箱 AI 助手集成开发计划

本计划旨在为压铸工具箱集成一个沉浸式、高可配置的 AI 助手模块。该模块将作为“压铸专家”角色，通过兼容 OpenAI 协议的接口与 LLM 服务交互，支持流式响应、Markdown/LaTeX 渲染及 Token 统计，且所有数据（配置、聊天记录）均保存在本地项目文件中。

## 1. 核心需求概览

*   **界面**：左侧侧边栏底部常驻入口，点击后进入全屏/沉浸式对话界面。
*   **角色**：默认预设为“压铸工艺专家”。
*   **兼容性**：支持自定义 API Base URL 和 API Key，兼容所有 OpenAI 接口格式（如 OpenAI, DeepSeek, LocalAI 等）。
*   **存储**：后端（Node.js）管理配置与聊天记录，数据持久化存储在 `server/data/` 下的 JSON 文件中。
*   **交互**：打字机流式响应 (Streaming)。
*   **渲染**：支持 Markdown（表格、列表、代码块）及数学公式 (LaTeX)。
*   **统计**：自动统计并展示 Token 消耗量。

---

## 2. 技术架构与数据设计

### 2.1 数据存储 (JSON)

所有数据文件存放在 `server/data/` 目录：

1.  **`ai_config.json`** (配置信息)
    *   存储供应商标识、API 基础地址、密钥（明文或简单加密）、默认模型、系统预设提示词以及上下文限制数量。

2.  **`ai_chats.json`** (会话记录)
    *   存储会话列表，包含每个会话的 ID、标题、创建时间、更新时间、总 Token 消耗。
    *   存储具体的对话消息内容，包括角色（用户/助手）、内容、时间戳以及单条回复的 Token 估算。

### 2.2 后端 API (`server/index.js`)

我们需要扩展现有的 Express 服务器，增加以下路由：

*   **配置管理**：提供获取和更新 AI 配置的接口。
*   **会话管理**：提供获取会话列表、创建新会话、删除会话的接口。
*   **核心流式接口**：
    *   接收用户消息、会话 ID 和模型参数。
    *   处理流程：加载历史记录 -> 拼接 System Prompt -> 请求外部 API -> 流式返回给前端 -> 异步保存记录与 Token 统计。

## 3. 前端开发任务 (React + Ant Design)

### 3.1 路由与入口
*   **路由**：新增独立路由页面。
*   **入口组件**：修改侧边栏菜单，在底部固定添加“AI 助手”按钮。

### 3.2 页面布局
采用**双栏布局**：
*   **左侧 (260px)**：历史会话列表。
    *   分组显示（今天、昨天、更早）。
    *   新建对话按钮。
    *   设置按钮（打开配置模态框）。
*   **右侧 (Flex-1)**：对话主区域。
    *   **顶部**：当前模型显示、Token 统计概览。
    *   **中间**：消息滚动视图。
        *   用户气泡：品牌色背景。
        *   AI 气泡：支持 Markdown 和 LaTeX 数学公式。
        *   代码块：语法高亮 + 复制按钮。
    *   **底部**：输入区域。
        *   支持快捷键换行。
        *   发送/停止按钮。

### 3.3 状态管理
*   管理当前会话状态、加载状态和流式接收的数据，避免 props 层层传递。
